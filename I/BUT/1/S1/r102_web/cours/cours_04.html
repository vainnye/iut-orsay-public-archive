Placement des Eléments

Introduction

  • Le placement n'est pas un problème simple

  • En CSS, tout élément a une boîte qui l'englobe
    Chaque élément n'est pas une boîte, mais en remontant le DOM, une boîte le contient → box model

  • La plus grande boîte contient l'élément html
    Elle s'étend sur toute la largeur disponible, et la hauteur nécessaire

  • A l'intérieur le placement débute en haut à gauche
    Le flot des éléments vient la remplir (⤳ écriture d'une suite de mots)

  • Les sous-blocs correspondent à des sous-boîtes. Approximativement :

    1. Un saut de ligne (que la ligne courante soit ou non pleine)

    2. Création d'une boîte imbriquée (pleine largeur)

    3. Remplissage récursif (selon élément ; i.e. selon sous-arbre du DOM)

Squelette d'une boîte

  • Une boîte est composée de 4 morceaux :

    • content-box : affiche le contenu de l'élément (sous arbre DOM)
      en css, sa taille est fixée par width et height par défaut,
      ou par inline-size et block-size avec writing-mode

    • padding-box : espacement autour du content
      en css, sa taille est gérée par padding

    • border-box : bordure autour du padding
      en css, son aspect et sa taille sont gérés par border

    • margin-box : espacement autour de border un "vide" autour de la bordure
      en css, sa taille est gérée par margin

  • schéma

Quelle taille fait ma boîte ?

  • Pour calculer la taille, il faut tenir compte de toutes les composantes

  • MAIS les unités des composantes ne sont pas forcément les mêmes
    Par exemple width:25%; et border:2px solid black;

  • CSS nous offre la fonction calc() qui sait gérer ces mélanges

  • MAIS, les arrondis des navigateurs vont poser problème

  • En utilisant box-sizing:border-box; le calcul de width et height inclut le padding et le border

  • Malheureusement, margin et padding peuvent "disparaitre" !

    • pour padding, dans le cas inline il ne compte pas toujours (cf outer display)

    • pour les margin, il y a le mécanisme de collapse

  • Là, on finit par atteindre la limite de la patience des web developpers....

Relation avec les boîtes autour : outer display

    • block (div, h1, p … [[exemple 00]] )

      • la boîte est un rectangle

      • la boîte débute sur une nouvelle ligne

      • on peut modifier sa longueur et sa largeur (par défaut, pleine largeur)

    • inline (span, a, em, strong … [[exemple 01]] )

      • la boîte est un ruban

      • pas de nouvelle ligne

      • on ne peut pas modifier sa longueur ou sa largeur

      • (top et bottom) (padding et margin) n'affectent pas les autres inline

      • (left et right) (padding et margin) affectent les autres inline

    • inline-block (définir à la main dans le CSS [[exemple 01]] )

      • la boîte est un rectangle

      • pas de une nouvelle ligne

      • on peut modifier sa longueur ou sa largeur (par défaut, largeur contenu)

Disposition des boîtes internes : inner display

    • normal flow (par défaut)

      • Les boîtes internes respectent leur outer display

    • flex (définir à la main dans le CSS)

      • amélioration de la mise en page flot
        les boîtes se placent les unes à ma suite des autres

    • grid (définir à la main dans le CSS)

      • amélioration de la mise en page tableau
        les boîtes sont disposées sur une grille

Sortie du flux : float

  • La propriété float modifie le comportement d'une boîte
    par rapport à ses voisins (à sibling plus précisement)
    On obtient une sorte d'encart

  • Une boîte peut être placée en float:right ou float:left

  • Elle est alors sortie du flot de ses siblings
    et tassée prioritairement à droite ou à gauche (dans la boîte parent)

  • Le reste du contenu est ensuite placé selon le calcul de flot
    pour remplir la place vacante (dans la boîte parent)

Positionnement arbitraire d'une boîte

  • La propriété css position permet de sortir de la logique de flot

    • static : ne modifie pas le positionnement
      Cependant l'élément devient positionned
      (Il pourra servir de référent aux positionnements emboités)

    • relative : permet de décaler l'élément, par rapport à la position par défaut
      Le décalage est défini via les propriétés left, right, top et bottom
      Un trou est laissé à sa position par défaut

    • absolute : L'élément est retiré du flot, sans laisser de trou
      Il est placé relativement à son premier parent positionned (via left, …)

    • fixed : Comme absolute, et non affecté par le scrolling

    • sticky : Initialement, l'élément a sa position par défaut
      Lors du scrolling, les quatres propriétés left, …
      sont utiliées comme bornes sur la position atteignable

  • Les quatre dernières valeurs empilement sur le z-index,
    plaçant l'élément au-dessus de son flot d'origine    [[exemple 03]]

Flexbox

Flexbox : amélioration de placement par flot

  • La flexbox est un inner display qui porte la déclaration display:flex;
    Cette boîte sert de conteneur (flex container)
    pour organiser d'autres boîtes (flex item)

  • C'est une boîte dans laquelle ses fils directs sont placés selon un flot
    Son rôle est d'améliorer la gestion :

    • de la taille des fils (grow, shrink, stretch)

    • des espacements (justify-content, align-items, gaps)

    • du passage à la ligne (flex-wrap)

    • de la direction du flot (float-direction)

  • Elle introduit un saut conceptuel car l'ordre d'affichage des fils
    n'est plus nécessairement celui du DOM (propriété order)

  • Consultez le tutoriel est proposé par css-tricks et le poster l'accompagnant

Grid

Grid : amélioration du placement par tableau

  • Le grid est aussi un inner display déclaré par display:grid;
    Donc on a un conteneur qui organise d'autres boîtes
    Il améliore le placement en tableau

  • Son rôle est d'améliorer la gestion
    de la taille des fils
    des espacements

  • En premier décrit la matrice des cases
    Puis chaque fils directs est placé dans une case
    ou un rectangle (c.f. colspan / rowspan)

  • grid et flexbox, décorrèlent donc affichage et ordre dans le DOM

Grid : amélioration du placement par tableau(suite)

  • La définition du tableau (ou grille) peut se faire en explicitant

    • la forme de chaque ligne (nb/taille de cases), qui se répétera

    • la forme de chaque colonne (idem), qui se répétera

    • les deux, qui combineront leurs propriétés

    • et, éventuellement, en ajoutant un nommage des zones (cases ou rectangles)

  • Si les zones sont nommées → Les fils sont placés en indiquant la zone

  • Sinon, la définition de la grille
    induit la définition de traits de séparation des lignes/colonnes

  • Les fils sont alors placés en décrivant la zone qu'ils doivent remplir
    grâce aux traits qui la délimitent

  • Consultez le tutoriel est proposé par css-tricks et le poster l'accompagnant

Responsive design

Responsive : Que veut-on ?

  • Le Oxford dictionnary définit
    Responsive : reacting quickly and positively

  • Les clients l'interprètent comme
    Qui va bien, pour tout le monde, tout le temps

  • Bref.. le premier des critères est d'avoir un document qui s'affiche correctement en fonction de la taille d'écran disponible

  • C'est déjà très ambitieux, et on se limitera le plus souvent à 3, voire 4 formats :
    téléphone, tablette, ordi, télé

  • Comme on ne sait même pas les détecter à coup sûr, on procèdera usuellement par un découpage en intervalles selon la largeur de l'écran

Responsive, avec nos outils de mise en page

  • flexbox et grid gérant mieux tailles et positionnements
    on tentera de se reposer sur eux, autant que possible

  • La directive @media permet de tester la largeur d'écran disponible
    pour choisir quelle version du css activer. Par exemple : 

@media (max-width: 600px) {
  p{ font-size : 2rem; }
  /* … */
}

@media (min-width: 601px) 
  and (max-width: 1200px) {
    p{ font-size : 1.2rem; }
    /* … */
}

@media (min-width: 1201px) 
  and (max-width: 2000px) {
    p{ font-size : 1rem; }
    /* … */
}

@media (min-width: 2001px) {
  p{ font-size : 1.5rem; }
  /* … */
}

Responsive, avec nos outils de css

  • Une technique pratique consiste à combiner
    l'importation de feuille css (@import)
    et les media query (@media)

    1. On traite la mise en page correspondant aux différentes conditions
      dans différents fichiers (par ex mobile.css tablette.css, …)

    2. On introduit un fichier intermédiaire (par exemple style.css)
      seule feuille de style liée dans le document html

    3. Dans ce fichier css, juste après l'encodage,
      on gère les importations effectives avec par mediaquery

    Par exemple : 

        @charset "utf-8";
        @import url("./mobile.css")   (max-width: 600px);
        @import url("./tablette.css") (min-width: 601px) and (max-width: 1200px);
        ...
        /*remarquez la syntaxe simplifiée, sans @media */

R1.02 - HTML / CSS - TP 4

Dans ce TP vous allez tester les techniques de placement float, flexbox, et grid. Vous rendrez ensuite vos pages responsive, afin qu'elles s'adaptent à trois largeurs d'écran classiques.

  1. Mise en place

      1. Créez un répertoire tp4. Copiez les éléments de base dans ce répertoire (fichier base.zip à décompreser).

      2. Dans /styles, créez trois nouveaux fichiers ne contenant que la déclaration d'encodage : float_101.css, flexbox_101.css, et grid_101.css

  2. Pas à pas
    1. Float

      Pour cette question, sauf indication contraire, le travail de style est à faire dans float_101.css

      1. Dans le fichier index.html ajoutez le lien vers le fichier float_101.css

      2. Modification des titres principaux :

        1. Pour chaque titre de section, remplacez la première lettre par une image (voir /images/alphabet). (éventuellement, encadrez le tout par un div, pour vous aider ; mais ce n'est pas indispensable). Mettez en commentaire la règle qui gérait le titre dans index.css

        2. Décollez le titre de la gauche. Encadrez l'image en doré (GoldenRod). Positionnez l'image à gauche, avec float

        3. Retravaillez le style de l'élément h1 en redéfinissant sa taille et le soulignement (ceci remplace ce qui existait précédemment dans index.css)

        4. Avec la même couleur dorée, ajoutez un soulignement de toute la ligne de titre, en utilisant une bordure basse (border-bottom)

      3. Modification des titres secondaires :

        1. Pour chaque titre d'article, supprimez le numéro hiérarchique (par exemple "1.2" dans "1.2 In sed sapien") et ajoutez une image (voir /images/chiffres) indiquant son numéro simple (seulement 2 pour cet exemple)
          (Attention : ne rajoutez pas de div cette fois-ci)
          Supprimez la règle qui gérait le titre dans index.css (utilisez les commentaires, plutot qu'une suppresion définitive)

        2. Ajoutez une règle css pour encadrer l'image en doré plus foncé (DarkGoldenRod). Positionnez l'image à droite, avec float

        3. Retravaillez le style de l'élément h2 en redéfinissant sa taille et le soulignement (comme précédemment dans index.css)

        4. Avec la même couleur, ajoutez un soulignement de toute la ligne de titre, en utilisant une bordure basse

        5. Remarquez que cette bordure démarre trop tôt. Retaillez-la en supprimant cet overflow. (remarque : cela pourrait aussi servir pour les h1)

      4. On veut tester le comportement avec des blocs float de plus grande taille. Plutôt que de rechercher des images plus grandes, augmentez la taille des images précédemment ajoutées grâce au css : Utilisez l'inspecteur de Firefox pour faire varier la taille des images à la volée, et observer le comportement des textes. Après vous expériences, revenez à une taille raisonnable.

      5. On veut tester le comportement avec plusieurs blocs float en compétition. Pour cela, modifiez l'élément aside : on veut obtenir un rectangle situé à gauche, sur 40% de la largeur de la page.
        De même, une fois vos tests faits, supprimez ces modifications.

      6. Attention :

        Vous avez supprimé deux informations du fichier index.css : le formatage des h1 et celui des h2. Il faut donc rajouter dans les autres pages (tableau et formulaire) des liens vers le css qui contient maintenant les informations de style (float_101).

        Mais ce n'est pas tout. Vous avez aussi changé la structure pour les titres, en ajoutant une image (et peut être un div). Il faut donc aussi répercuter ces modifications dans les autres fichiers (tableau.html,…).

    2. FlexBox

      Pour cette question, sauf indication contraire, le travail de style est à faire dans flexbox_101.css

      1. Nous allons tester les flexbox sur l'affichage d'une collection d'images (voir /images/collection). Créez pour cela une nouvelle page images.html :

        1. Elle doit contenir la même structure de base que les autres (header, nav, main, footer, liens vers les fichiers css, …)

        2. Elle doit s'intégrer à la navigation, et ce, dans toutes les pages

        3. Son main ne contiendra qu'une section, avec un bloc égrainant toutes les images : 

          <div class="album">
	<img src="./images/collection/000.webp" />
	<img src="./images/collection/001.jpeg" />
	<img src="./images/collection/002.webp" />
	<img src="./images/collection/003.jpg" />
	<img src="./images/collection/004.jpg" />
	<img src="./images/collection/005.png" />
	<img src="./images/collection/006.jpg" />
	<img src="./images/collection/007.jpg" />
	<img src="./images/collection/008.jpg" />
	<img src="./images/collection/009.webp" />
	<img src="./images/collection/010.avif" />
	<img src="./images/collection/011.jpg" />
	<img src="./images/collection/012.jpeg" />
	<img src="./images/collection/013.webp" />
</div>

      2. Ajoutez dans images.html le lien vers flexbox_101.css

      3. Dans flexbox_101.css, retaillez les images, en leur donnant une fine bordure noire arrondie.

        Maitrisez la taille des images, en limitant la largueur à 90% de celle de son conteneur (le div) et la hauteur à 15rem

        Faites varier la taille de la fenêtre de Firefox. Vérifier le bon comportement de l'album.

      4. Passez l'album en mode flexbox

      5. Evitez le débordement, en demandant à l'album de passer à la ligne lorsque ça sera nécessaire

      6. Forcer l'espacement des images sur chaque ligne, en demandant la répartition de la place disponible

      7. Rajoutez des séparations entre les lignes

      8. Remarquez que la première et la dernière ligne n'ont pas été séparées des bords haut et bas. Forcez ces espacements

      9. Faites varier la taille de la fenêtre de Firefox. Vérifier le bon comportement de l'album.

      10. Retravaillez le style de l'album (bordure, fond, …) pour le rendre un peu plus esthétique

    3. Navigation responsive en flexbox

      Vous avez pu constater que flexbox aide à gérer la largeur disponible. Nous allons utiliser cela pour refaire la barre de navigation.

      1. Créez un nouveau fichier flexbox_101_nav.css dans le répertoire adéquat, contenant l'annonce de base d'encodage. Ajoutez un lien vers ce css dans toutes les pages. Désactivez la gestion de la navigation dans le fichier index.css

        Dans la suite, sauf mention contraire, on travaille dans flexbox_101_nav.css

      2. Réactivez le style de base de la navigation, à savoir ceux définissant :       nav a       nav a.ici       nav a:hover

      3. Nous voulons que flexbox gère la taille : Dans nav a, retirez le bricolage, à savoir :
        "display : inline-block;" et "width : xx%;", puis passez en flexbox

      4. Répartissez les liens sur la ligne

      5. Testez en faisant varier la largeur de la fenêtre Firefox. Gérez les débordements constatés (rajoutez aussi les espacements nécessaires)

      6. Remarquez que la navigation s'est collée au main. Séparez-la un peu pour que ce soit plus propre.

    4. Grid

      Pour cette question, sauf indication contraire, le travail de style est à faire dans grid_101.css

      1. Dans index.html, nous allons utiliser un grid pour organiser les article en simulant des colonnes. Cela ne nécessite pas de retoucher la structure du dom. Il suffit d'utiliser le parent des article comme container naturel. Pour cela, modifiez la propriété adéquate pour le faire passer en grid

        Répartissez les fils en quatre colonnes de largeur équivalentes

      2. Nous sommes gênés par le titre et son image, qui n'entrent pas dans la logique de colonne que l'on veut appliquer. Par chance, ces deux éléments sont déjà regroupés dans un div. Ciblez-le, et utilisez la propriété grid-column-start pour gérer ce cas particulier

      3. Testez en faisant varier la largeur de la fenêtre Firefox.

        Clairement, autant le flexbox utilisé pour la navigation fait bien son travail, autant les petites colonnes deviennent désagréable à lire.

        Utilisez les mediaquery pour que le nombre de colonnes soit réduit à 2 lorsque la largeur passe sous les 1200px, puis à une seule colonne lorsque la largeur passe sous les 600px.

  3. Valider le html et le css

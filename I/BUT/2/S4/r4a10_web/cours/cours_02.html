Bases du langage

Types primitifs

  • Boolean   ≜   deux constantes    true et false
    ⚠ falsy et truthy

  • Number   ≜   implémentation    un double
    plus quelques constantes :
    Infinity, -Infinity, NaN, Number.MAX_VALUE, Number.MIN_VALUE
    ⚠ overflow sur    +/- min/max    ⤳    +/- Infinity    ou bien    0
    ⚠ division par    +/- 0    ⤳    +/- Infinity    (au lieu de NaN)

  • BigInt (une classe)   ≜   entiers, sans limite de taille

  • String (une classe)   ≜   plein de méthodes utiles, mais ⚠ retro-compatibility hazards : 

    console.log( 2 + "0.5"); ⤳ "20.5"
console.log( 2 * "0.5"); ⤳ 1

Types primitifs (suite)

  • Undefined   ≜   une unique constante    undefined    (absence of a value)

  • Null   ≜   une unique constante    null    (absence of an object)
    ⚠ nullish    (⇒ undefinedish)

  • Symbol : (hors sujet pour ce cours)

String template

Une facilité pout formater des chaînes de caractères

  • Une string template est un littéral de String (comme 4.5 est un littéral pour Number)

  • Syntaxe : suite de caractères comprise entre un ` de début et un de fin
    ( ⇒ ne contient pas ce délimiteur → `\`` )

  • Particularités :

    • multiline : Peut contenir des passages à la ligne (plus besoin de \n)

    • interpolation : Peut contenir des sous-suites de caractères délimitées par ${ et }
      Chaque sous-suite est évaluée en javascript (expression)
      La valeur obtenue est utilisée pour produire la String

    • tagged : Peut-être préfixée par un nom de fonction

Deux types intégrés : dict et array

  • Le langage intègre dans sa syntaxe deux types presque classiques :
    sugar coating → le langage s'adapte pour faciliter leur utilisation

  • les dictionnaires
    { ... __cle__ : __valeur__, ... }

  • les tableaux
    [ ... __valeur__, ... ]

  • le JS dans json, c'est pour javascript 💡

les dict

  • {} est un littéral d'Object (comme 3 est un littéral de int)

  • Les clés peuvent être des strings (ou mieux si ce sont des identifiants)
    elles peuvent aussi être des ints, ou un mélange du tout (str|id|int)

  • Les syntaxes admises sont : o[k] et o.k (si k ≡ identifiant)

  • Les clés peuvent être associées à des valeur classiques ou à des fonctions
    → on a donc la poo gratuit

Les tab

  • [] est un littéral d'Array

  • C'est un tableau, indexé de 0 à length-1

  • Il est resizable (au-delà du raisonnable) 

    let t = [];  ⤳ []
t[0] = "a";  ⤳ ["a"]
t[4] = "e";  ⤳ ["a", undefined, undefined, undefined, "e"]
delete t[2]; ⤳ ["a", undefined, undefined, undefined, "e"]

  • il est aussi utilisable comme liste / pile / file grâce aux méthodes :
    .push()   .pop()   .shift()   .unshift()   …

Attention dict et tab méchants

  • Les deux sont des Object : 

    console.log( typeof {} ); ⤳ "object"
console.log( typeof [] ); ⤳ "object"

  • L'interpolation des String peut interférer : 

    if( '1'   == 1 ) console.log( 'aille' ); ⤳ "aille"
if( '0x1' == 1 ) console.log( 'aille' ); ⤳ "aille"

    → d'où :

    t.un   = "1";  console.log( t ); console.log( t.un );
  ⤳ ["a", undefined, undefined, undefined, "e"]   
  ⤳ "1"
t['1'] = "b";  console.log( t ); console.log( t['1'] );
  ⤳ ["a", "b",       undefined, undefined, "e"]   
  ⤳ "b"

Attention dict et tab méchants (suite)

  • → ou aussi :

    let o = {}; o.un = 1; 
console.log( o['1'] );
  ⤳ undefined

o[1] = 'a'; 
console.log( o );
  ⤳ { 1: "a", un: 1 }
console.log( o['1'] ); console.log( o[1] ); 
  ⤳ "a"
  ⤳ "a"

JSON

  • Syntaxe des littéraux ⇒ définition facile d'objets à la volée ⇒ JSON

  • Attention : json est plus contraint que la syntaxe javascript

    • ⚠ pas de méthodes

    • 'primitifs' ↔ terminaux ≜ nombres, strings, true, false, null

    • dict / tab

    • clés ≜ des strings bien formées

    • + imbrications de dict et tab

  • Sauvegarder / Lire / Transmettre un json ⇒ conversion :
    JSON.stringify( _obj_ )   et   JSON.parse( _str_ )

Instructions et déclarations de variables

  • Langage impératif : suite d'instruction exécutées dans l'ordre (ou presque)

  • ⚠ Instruction : terminée ou non par ; 

    return 
       0;

  • Déclaration de variables : 3 mots clés   var / let / const
    différence de portée (globale / bloc / bloc ) et qq subtilités 😓

    meilleur controle → strict mode
    (le fameux "use strict;" en début de fichier)

  • En fait …, var a une portée bloc-function …
    → passez toujours en mode strict
    → privilégiez let et const (schisme const>let vs. let>const)

Structures de contrôle : conditionnelles

  • if  ( __bool_like__ )  { ... }
elif( __bool_like__ )  { ... }
...
else                   { ... }

  • ⚠ Utilisez les tests forts :   ===   et   !==

  • ⚠ Les opérateurs   ||   et   &&   ne sont pas des fonctions bool x bool → bool
    mais des court-circuits T x ? → T|? 

  • switch( __exp__ ){
  case __val__ :  ...  break;
  ...
  default:        ...
}

Structures de contrôle : itérations while

  • Boucle tant que 

    while ( __bool_like__ )  { ... }

  • Boucle faire 

    do { ... } while ( __bool_like__ );

  • On peut utiliser continue et break (dans tous les types d'itérations)

  • L'utilisation de label peut permettre une sortie de boucles imbriquées
    (⚠ approche toujours déconseillée, et interdite dans ce cours)

Structures de contrôle : itération for

  • Boucle classique 

    for ( let _var_ = ... ; __bool_like__ ; ... )  { ... }

  • Boucle sur les valeurs d'un itérable (Array, String, collections) 

    for ( const _elem_ of _iterable_ )  { ... }

  • Boucle sur les propriétés (clés) d'un Object 

    for ( const _prop_ in _obj_ )  { ... }

Structures de contrôle : itération for (suite)

  • Donc :

    let t = [ 'a', 'b', 'c' ];
for ( const p of t )  { console.log( p ); } ⤳ a b c
for ( const e in t )  { console.log( e ); } ⤳ 0 1 2

let o = { '0':'a', '1':'b', '2':'c' };
for ( const p of o )  { console.log( p ); } 
	⤳ TypeError: o is not iterable
for ( const e in o )  { console.log( e ); } ⤳ 0 1 2

  • Utilitaire associé : unpacking 

    let o = { '0':'a', '1':'b', '2':'c' };
for ( const [k,v] of Object.entries(o) )  
	{ console.log( `cle:${k}->val:${v}` ); }
⤳ "cle:0->val:a"  "cle:1->val:b"  "cle:2->val:c"

Structures de contrôle : itération .forEach()

  • Array dispose d'une méthode d'itération : .forEach( __cb__ )

  • → exécute __cb__ sur chaque élément du tab

  • Cette fonction reçoit en premier paramètre, un élément à traiter

  • Optionnellement, le second argument indique son indice

  • ⚠ contexte DOM
    .querySelectorAll() ne retourne pas Array
    mais un objet ayant un .forEach( __cb__ )

    → on écrira donc : 

    let res = document.querySelectorAll( __sel__ );
//puis
res.forEach( (__item__)       => { ... } );
//ou
res.forEach( (__item__,__idx__) => { ... } );

  • Introduisez des fonctions utilitaires 

    function __nom__( __param__, ... ){ .... return ...; }

    (plus de détails au cours 3)

  • A partir d'un element du dom, i.e. un noeud de l'arbre,
